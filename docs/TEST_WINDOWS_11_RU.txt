SacramentuOS 0.2.3 — проверка на Windows 11
=============================================

Рекомендуемый запуск: напрямую через Multiboot-kernel, без ISO/GRUB.
Так меньше проблем с зависанием SeaBIOS на строке Booting from DVD/CD.

1. Открой PowerShell от администратора и установи WSL:

   wsl --install

2. Перезагрузи Windows, открой Ubuntu/WSL и поставь зависимости:

   sudo apt update
   sudo apt install -y build-essential gcc-multilib make qemu-system-x86

3. Перейди в папку проекта. Пример, если архив распакован в Downloads:

   cd /mnt/c/Users/<ТВОЙ_ПОЛЬЗОВАТЕЛЬ>/Downloads/SacramentuOS

4. Быстрый запуск готового ядра:

   make run-kernel

5. Полная пересборка из исходников и запуск:

   make clean
   make CC=gcc run-kernel-built

Что должно произойти
--------------------

1. Появляется boot sequence.
2. Перед большим логотипом экран очищается.
3. Логотип SacramentuOS отображается зеленым цветом по умолчанию.
4. Система просит выбрать username.
5. После ввода имени prompt становится таким:

   username@sacramentuOS>

Основные проверки
-----------------

Введи по очереди:

   help
   docs
   commands
   whoami
   fetch
   color amber
   help
   color green
   set USER tester
   whoami
   env

Проверка: help/docs должны быть плотными, без большого количества пустых строк. Prompt должен менять имя и цвет.

Проверка Ctrl+C
---------------

1. Запусти длительную команду:

   sleep 10000

2. Нажми Ctrl+C.

Ожидаемо: команда завершается, появляется сообщение cancelled by Ctrl+C, shell не зависает.

Проверка редактора кода
-----------------------

1. Открой редактор:

   edit main.c

2. Введи небольшой код:

   int main() {
       return 0;
   }

3. Проверь перемещение:

   стрелка вверх/вниз      переход между строками
   стрелка влево/вправо    переход по символам
   Home/End                начало/конец строки
   Backspace/Delete        удаление символов
   Enter                   новая строка

4. Горячие клавиши:

   F7   сохранить файл в RAM и остаться в редакторе
   F8   сохранить файл в RAM и выйти в shell
   F9   выйти без сохранения
   Ctrl+C тоже выходит из редактора без сохранения

5. После F8 проверь файл:

   files
   show main.c
   grep return main.c
   head main.c
   wc main.c

6. Удали RAM-файл:

   rm main.c
   files

Важно
-----

Редактор безопасный: он работает только в RAM, использует фиксированные буферы и не пишет на диск. Ограничения: 8 файлов, 64 строки в файле, 119 символов в строке.
