SacramentuOS 0.2.1 — исправление зависания на Booting from DVD/CD...
=====================================================================

Проблема
--------
Если QEMU показывает:

    Booting from Hard Disk...
    Boot failed: could not read the boot disk
    Booting from Floppy...
    Boot failed: could not read the boot disk
    Booting from DVD/CD...

и дальше ничего не происходит, значит проблема не в shell-командах SacramentuOS, а в цепочке загрузки ISO/GRUB внутри QEMU/WSL/Windows.
Я добавил обходной и более надежный способ запуска: загрузку ядра напрямую через QEMU -kernel.
Он не использует ISO и GRUB, поэтому не зависает на этапе DVD/CD.

Быстрый рабочий запуск в WSL
----------------------------
1. Распакуй архив проекта.
2. Открой Ubuntu/WSL.
3. Перейди в папку проекта. Пример:

    cd /mnt/c/Users/<ТВОЙ_ПОЛЬЗОВАТЕЛЬ>/Downloads/SacramentuOS

4. Установи QEMU, если он еще не установлен:

    sudo apt update
    sudo apt install -y qemu-system-x86

5. Запусти готовое ядро напрямую:

    make run-kernel

Или вручную:

    qemu-system-i386 -machine pc -m 256M -serial stdio -no-reboot -no-shutdown -kernel prebuilt/sacramentuos.kernel

Полная проверка со сборкой исходников
-------------------------------------
Если нужно проверить, что система собирается из исходников:

    sudo apt update
    sudo apt install -y build-essential gcc-multilib make qemu-system-x86
    make clean
    make CC=gcc run-kernel-built

Это соберет build/sacramentuos.kernel и сразу запустит его напрямую.

Запуск из Windows PowerShell без WSL-сборки
------------------------------------------
Если установлен QEMU for Windows, можно запускать готовое ядро прямо из PowerShell:

    powershell -ExecutionPolicy Bypass -File .\scripts\run_qemu_windows.ps1

Если QEMU установлен не в стандартную папку:

    powershell -ExecutionPolicy Bypass -File .\scripts\run_qemu_windows.ps1 -QemuPath "C:\Program Files\qemu\qemu-system-i386.exe"

Проверка ISO/GRUB, если нужен именно ISO
---------------------------------------
ISO-режим оставлен, но он зависит от корректной установки GRUB BIOS-модулей и xorriso.
Для WSL/Ubuntu установи:

    sudo apt update
    sudo apt install -y grub-pc-bin grub-common xorriso mtools qemu-system-x86

Потом собери и проверь ISO:

    make clean
    make iso-prebuilt
    make check-iso
    qemu-system-i386 -machine pc -m 256M -boot d -cdrom build/SacramentuOS.iso -serial stdio -no-reboot -no-shutdown

Если снова зависает на Booting from DVD/CD..., используй прямой запуск:

    make run-kernel

Что изменено в версии 0.2.1
---------------------------
1. Добавлен надежный запуск без ISO/GRUB:

    make run-kernel
    make run-kernel-built

2. Добавлен принудительный CD/DVD boot для ISO:

    make run-iso
    make run-prebuilt

3. GRUB-меню больше не скрывается мгновенно. Теперь timeout = 5 секунд.
   Это помогает понять, дошла ли загрузка до GRUB.

4. В grub.cfg добавлены диагностические сообщения:

    Loading SacramentuOS kernel...
    Starting kernel...

5. Добавлен PowerShell-скрипт:

    scripts/run_qemu_windows.ps1

6. Добавлены Linux/WSL-скрипты:

    scripts/run_qemu_kernel.sh
    scripts/run_qemu_iso.sh

Минимальный тест внутри SacramentuOS
------------------------------------
После загрузки должны появиться логотип и строка приглашения.
Введи команды:

    ver
    fetch
    commands
    help top
    set user Fernando
    get user
    note add test note
    notes
    todo add boot test
    todo list
    done 1
    hash hello
    rot13 hello
    edit main.c
    files
    show main.c
    cat /etc/os-release
    grep OS /etc/os-release
    top
    kernel
    log

Ожидаемый результат
-------------------
При make run-kernel экран SeaBIOS может мелькнуть, но система должна перейти в SacramentuOS без зависания на DVD/CD.
Если появляется shell и работают команды ver/commands/top — запуск исправлен.
