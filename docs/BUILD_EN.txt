SacramentuOS — build and run guide
==================================

What it is
----------
SacramentuOS is a tiny educational x86/i386 console operating system.
It is written in C and GNU Assembly, boots through GRUB with Multiboot v1,
runs in VGA text mode 80x25, and includes a PIT timer, IRQ keyboard input and a built-in shell.

It is not a Linux/Windows replacement and not a full modern OS. It is a working starter OS project
that can be launched in QEMU and written to a USB flash drive as an ISO image.

Limitations
-----------
1. Main boot mode: BIOS/Legacy/CSM. Some UEFI-only machines may not boot it from USB.
2. Kernel keyboard layout: US QWERTY.
3. The OS filesystem is virtual/demo-only: ls/cat show built-in pseudo-files.
4. Writing the ISO to USB overwrites the selected drive. Check the device name carefully.

Project layout
--------------
boot/grub/grub.cfg        — GRUB menu
include/                  — kernel headers
src/boot.S                — Multiboot header and Assembly entry point
src/interrupt.S           — ISR/IRQ stubs
src/*.c                   — kernel, VGA, keyboard, timer, shell, RTC, string utils
linker.ld                 — linker script
Makefile                  — kernel + ISO build
scripts/run_qemu.sh       — quick Linux QEMU launcher
scripts/write_usb_arch.sh — safer wrapper for writing the ISO to USB on Arch/Linux

SacramentuOS shell commands
---------------------------
help, clear, fetch, sysinfo, neofetch, about, echo, uname, whoami, mem, uptime,
ticks, date, time, calc, color, colors, theme, banner, logo, history, len, upper,
lower, reverse, repeat, sleep, rand, ascii, hexdump, ls, cat, pwd, motd, ver,
credits, matrix, prompt, reboot, halt.

Build on Arch Linux
-------------------
1. Install dependencies:

   sudo pacman -S --needed base-devel qemu-system-x86 grub xorriso mtools git

2. Install i686-elf-gcc and i686-elf-binutils.
   The easiest way is an AUR helper, for example yay:

   yay -S i686-elf-gcc i686-elf-binutils

   If you do not use an AUR helper, build those AUR packages manually.
   Host gcc can also work in some setups, but a cross-compiler is more reliable.

3. Enter the project directory:

   cd SacramentuOS

4. Build the ISO:

   make clean
   make iso

   Output:

   build/SacramentuOS.iso

5. Run in QEMU:

   make run

   or manually:

   qemu-system-i386 -cdrom build/SacramentuOS.iso -m 256M -serial stdio

Write to USB on Arch/Linux
--------------------------
WARNING: this overwrites the selected disk. Do not pass a partition such as /dev/sdb1.
Pass the whole device, for example /dev/sdb.

1. Find the USB drive:

   lsblk

2. Write the ISO:

   ./scripts/write_usb_arch.sh /dev/sdX

   Example:

   ./scripts/write_usb_arch.sh /dev/sdb

3. Reboot and choose the USB device in your boot menu.
   If your computer is UEFI-only, enable Legacy/CSM if available.

Build on Windows
----------------
Recommended method: WSL2. OS building needs Linux tools such as GRUB and xorriso.

Option A: Windows + WSL2 Ubuntu

1. In Administrator PowerShell:

   wsl --install

2. Reboot, then open Ubuntu in WSL.

3. Install dependencies:

   sudo apt update
   sudo apt install -y build-essential gcc-multilib qemu-system-x86 grub-pc-bin grub-common xorriso mtools make

4. Enter the project directory. Example if it is in Downloads:

   cd /mnt/c/Users/<YOUR_USER>/Downloads/SacramentuOS

5. Build the ISO with host gcc:

   make clean
   make CC=gcc iso

6. Run QEMU from WSL:

   qemu-system-i386 -cdrom build/SacramentuOS.iso -m 256M -serial stdio

Option B: Windows + WSL2 Arch

1. Install WSL2 and ArchWSL.
2. In ArchWSL:

   sudo pacman -Syu
   sudo pacman -S --needed base-devel qemu-system-x86 grub xorriso mtools git
   yay -S i686-elf-gcc i686-elf-binutils

3. In the project folder:

   make clean
   make iso
   make run

Write the ISO to USB on Windows
-------------------------------
1. Copy build/SacramentuOS.iso to a Windows folder.
2. Open Rufus.
3. Select your USB drive.
4. Select SacramentuOS.iso.
5. If Rufus asks for the write mode, choose DD mode.
6. Press Start. All data on the USB drive will be erased.
7. Reboot and select the USB drive in your boot menu.

Useful Makefile commands
------------------------
make clean     — remove build/
make iso       — build build/SacramentuOS.iso
make run       — build and launch QEMU
make check     — check Multiboot compatibility, if grub-file is installed

Troubleshooting
---------------
1. Check grub-mkrescue:

   grub-mkrescue --version

2. Check xorriso:

   xorriso -version

3. If grub-file is not found, install GRUB tools. The kernel can still be linked,
   but the Multiboot check will be skipped.

4. If i686-elf-gcc fails, try host gcc in WSL/Arch:

   make clean
   make CC=gcc iso

5. Always test the ISO in QEMU before writing it to a USB drive.
